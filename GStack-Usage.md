# GStack 使用指南

Garry Tan (YC CEO) 的 Claude Code 技能包，23 个专业角色 + 8 个工具。

## 核心工作流（推荐从这里开始）

```
/office-hours        → 描述你在做什么，Claude 会问 6 个关键问题
/plan-ceo-review     → CEO 视角战略评审：这个功能值得做吗？
/autoplan            → 自动生成实现计划
/review              → 有代码改动时跑代码审查
/qa                  → 测试你的页面
/ship                → 提交 PR
```

## 规划阶段

| 命令 | 说明 |
|------|------|
| `/office-hours` | 产品问诊，6 个关键问题审视产品想法 |
| `/plan-ceo-review` | CEO 视角战略评审，评估是否值得做 |
| `/plan-eng-review` | 工程经理视角技术方案评审 |
| `/plan-design-review` | 设计师视角 UI/UX 方案评审 |
| `/plan-devex-review` | 开发者体验评审 |
| `/autoplan` | 自动生成实现计划并执行 |

## 典型使用场景

| 场景 | 操作 |
|------|------|
| "修复 README 错字" | 直接说，不需要 gstack |
| "跑个安全审计" | 运行 `/cso` |
| "做个通知功能" | `/autoplan` → 实现 → `/ship` |
| "规划 v2 API 重构" | `/office-hours` → `/autoplan`，保存计划不实现 |

## 每个命令的完整场景示例

以下按一个完整的产品开发生命周期，自上而下演示每个命令的使用场景。

---

### 0. `/office-hours` — 产品问诊

**场景**：你有一个新功能想法——"给蓝海词筛选系统加一个 API 接口，让其他系统可以按关键词批量查询"

**你怎么做**：
```
/office-hours
然后说：我想给当前的蓝海词系统加一套 RESTful 批量查询 API，客户传关键词列表，返回每个词是否是蓝海词。
```

**Claude 会问你 6 个问题**：谁用、什么场景用、成功标准是什么、最大的风险是什么、为什么现在做、有没有更简单的做法。

---

### 1. `/plan-ceo-review` — CEO 战略评审

**场景**：经过 office-hours 讨论，你决定做批量查询 API，但需要确认这个功能是否值得投入

**你怎么做**：
```
/plan-ceo-review
然后说：批量查询 API 功能，预计 2 周开发。当前系统已有上传 Excel → AI 筛选的流程，现在要加一个同步 API 接口。
```

**Claude 会从商业角度分析**：这个功能带来多少收入？会拖慢主产品节奏吗？有没有更好的杠杆点？

---

### 2. `/plan-eng-review` — 工程技术评审

**场景**：CEO 评审通过了，现在需要工程师经理视角评估技术方案

**你怎么做**：
```
/plan-eng-review
然后说：方案是在现有 Spring Boot 项目里新增 /api/batch-query 接口，接收 JSON 数组，逐个调用 DashScope，返回结果。需要考虑并发限流、超时、缓存。
```

**Claude 会从工程角度评估**：技术栈选型、性能瓶颈、缓存策略、并发安全、是否需要引入消息队列。

---

### 3. `/plan-design-review` — UI/UX 设计评审

**场景**：API 功能需要在管理后台加一个可视化页面，让非技术人员也能用

**你怎么做**：
```
/plan-design-review
然后说：需要在现有的 index.html 管理页面加一个"批量查询"tab，用户粘贴关键词列表，点击查询，结果以表格展示。
```

**Claude 会从用户体验角度给建议**：布局、交互流程、空状态/错误状态处理、移动端适配。

---

### 4. `/plan-devex-review` — 开发者体验评审

**场景**：你要对外提供 API，需要考虑开发者接入体验

**你怎么做**：
```

然后说：批量查询 API 会暴露给第三方调用，需要写 API 文档、提供 SDK 示例。
```

**Claude 会审查**：API 命名是否一致、错误码是否规范、文档是否清晰、是否有认证鉴权机制。

---

### 5. `/autoplan` — 自动生成实现计划

**场景**：所有评审都通过了，现在要开始写代码

**你怎么做**：

### 17. `/cso` — 安全审计

**场景**：新 API 上线前，做一轮安全审查

**你怎么做**：
```
/cso
```

**Claude 会执行**：
- OWASP Top 10 检查：SQL 注入、XSS、CSRF、认证绕过
- STRIDE 威胁建模：仿冒、篡改、抵赖、信息泄露、拒绝服务、权限提升
- 特别检查：API Key 是否硬编码、文件上传是否有类型/大小限制

**Claude 会**：评估操作风险、确认备份策略、要求你确认每一步、如果有危险操作会阻止并警告。
